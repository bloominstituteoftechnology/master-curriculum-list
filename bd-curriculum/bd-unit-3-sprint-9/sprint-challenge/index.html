Sprint Challenge

Overview

This Sprint Challenge is designed to assess your understanding of the concepts covered in BD Unit 3 Sprint 9. You will apply your knowledge of design patterns, advanced OOP concepts, exception handling, and testing techniques in a comprehensive project.

Challenge Description

You will be building a Library Management System that demonstrates your understanding of advanced Java concepts. This system will track books, patrons, and borrowing transactions while implementing proper error handling and testing.

Requirements

  • Implement at least three design patterns from the ones you've learned
  • Create a proper class hierarchy with inheritance and polymorphism
  • Implement custom exception handling for different error scenarios
  • Write comprehensive unit tests for your implementation
  • Document your code using proper comments and follow Java coding standards

Project Structure

Your implementation should include the following components:

1. Model Classes

  • Book: Represents a book with attributes like title, author, ISBN, etc.
  • Patron: Represents a library patron with attributes like name, ID, etc.
  • Transaction: Represents a borrowing transaction

2. Service Classes

  • BookService: Manages operations related to books
  • PatronService: Manages operations related to patrons
  • TransactionService: Manages borrowing and returning operations

3. Exception Classes

  • BookNotFoundException: Thrown when a book is not found
  • PatronNotFoundException: Thrown when a patron is not found
  • BookNotAvailableException: Thrown when attempting to borrow an unavailable book

4. Design Patterns

  • Implement a Singleton for a central library database or configuration manager
  • Implement a Factory Method for creating different types of books or transactions
  • Implement an Observer for notification when a book's status changes
  • Implement any other pattern you find suitable for the application

5. Tests

  • Write unit tests for all service methods
  • Include tests for exception cases
  • Implement mocks where appropriate

Evaluation Criteria

Your submission will be evaluated based on the following criteria:

  • Design Pattern Implementation (25%): Proper implementation of at least three design patterns
  • OOP Principles (25%): Proper use of inheritance, encapsulation, and polymorphism
  • Exception Handling (20%): Comprehensive error handling with custom exceptions
  • Testing (20%): Thorough test coverage and proper test design
  • Code Quality (10%): Clean, readable code that follows Java conventions
