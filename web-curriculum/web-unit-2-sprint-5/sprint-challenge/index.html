Sprint Challenge

This sprint challenge will test your understanding of DOM manipulation, event handling, component creation, and network data consumption. You'll apply all the concepts you've learned throughout the sprint to build a complete web application.

Project Overview

For this sprint challenge, you'll create a web application that fetches data from an API and displays it using custom-built components. You'll implement event handlers, create reusable components, and handle network requests to build a complete, interactive application.

Project Requirements

Your application should include the following features:

  • Fetch data from a public API and display it on the page
  • Create reusable components for displaying the data
  • Implement event handlers for user interactions
  • Add search or filtering functionality
  • Handle loading states and errors gracefully
  • Use proper DOM manipulation techniques

Project Steps

  1. Set up your project structure and HTML skeleton
  2. Create component functions or classes for UI elements
  3. Implement event handlers for user interactions
  4. Fetch data from the API and handle responses
  5. Render the data using your components
  6. Add search/filter functionality and enhance the UI
  7. Test your application and fix any bugs

Grading Criteria

Your project will be evaluated based on:

  • Functionality - Does the application work as expected?
  • Code Quality - Is the code well-organized and maintainable?
  • Component Design - Are components reusable and well-structured?
  • Error Handling - Does the application handle errors gracefully?
  • User Experience - Is the interface intuitive and responsive?
  • Code Comments - Is the code well-documented?
