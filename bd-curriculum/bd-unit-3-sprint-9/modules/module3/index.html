Exception Handling in Detail

Exception handling is a critical aspect of creating robust Java applications. Proper error handling ensures that your programs can respond gracefully to unexpected conditions and provide meaningful feedback to users or other systems.

Java Exception Hierarchy:

  • Throwable: The root class of all errors and exceptions
    • Error: Serious problems that applications should not try to handle (e.g., OutOfMemoryError)
    • Exception: Problems that can be reasonably handled by applications
      • Checked Exceptions: Must be explicitly caught or declared (e.g., IOException)
      • Unchecked Exceptions (RuntimeException): Do not need to be explicitly caught (e.g., NullPointerException)

Exception Handling Techniques:

Basic try-catch-finally:

try {
    // Code that might throw an exception
    File file = new File("example.txt");
    FileReader fr = new FileReader(file);
    BufferedReader br = new BufferedReader(fr);
    String line = br.readLine();
    System.out.println(line);
} catch (FileNotFoundException e) {
    // Handle specific exception
    System.err.println("File not found: " + e.getMessage());
} catch (IOException e) {
    // Handle another type of exception
    System.err.println("Error reading file: " + e.getMessage());
} finally {
    // Code that always executes, regardless of exception
    System.out.println("Cleanup operations");
    // Close resources, etc.
}

Try-with-resources (Java 7+):

try (BufferedReader br = new BufferedReader(new FileReader("example.txt"))) {
    String line = br.readLine();
    System.out.println(line);
} catch (IOException e) {
    System.err.println("Error working with file: " + e.getMessage());
    e.printStackTrace();
}
// Resources automatically closed

Creating Custom Exceptions:

Custom exceptions allow you to create application-specific error handling that can provide more meaningful feedback.

// Custom checked exception
public class InsufficientFundsException extends Exception {
    private double amount;
    
    public InsufficientFundsException(double amount) {
        super("Insufficient funds: Attempted to withdraw " + amount);
        this.amount = amount;
    }
    
    public double getAmount() {
        return amount;
    }
}

// Usage
public class Account {
    private double balance;
    
    public void withdraw(double amount) throws InsufficientFundsException {
        if (amount > balance) {
            throw new InsufficientFundsException(amount);
        }
        balance -= amount;
    }
}

Best Practices for Exception Handling:

  • Only catch exceptions you can handle meaningfully
  • Don't swallow exceptions (catch and do nothing)
  • Clean up resources in finally blocks or use try-with-resources
  • Use specific exception types rather than catching Exception
  • Include helpful information in exception messages
  • Log exceptions properly for troubleshooting
  • Throw exceptions at the appropriate level of abstraction
